add +

« + » — adding element(s) to the collection. It is available for object and array elements.

In the case of object you add a key-value pair to the object, where key is a string and value is any JSON, including function.

In the case of array you add any JSON value to the array, including function. You can add multiple element to the array at once.

Examples:
  • {"key1": "value1"}   +   {"key2": "value2"}  =  {"key1": "value1", "key2": "value2"}
  • [1,2,3]   +   [4,"5",6]  =  [1,2,3,4,"5",6]
  • [1,2,3]   +  [[4,"5",6]]  =  [1,2,3,[4,"5",6]]

reset ~

« ~ » — reseting any node value to null. By double clicking.

Examples:
  • {"key1": "value1"}   ~   "key1"   =  {"key1": null}

remove -

« - » — completely removing the node. By double clicking.

Examples:
  • {"key1": "value1", "key2": "value2"}   -   "key1"  =  {"key2": "value2"}
  • ["first", "second", "third"]   -   "second"  =  ["first", "third"]

change value and type

— is an input field, where you can input any JSON data. The directive automatically determine the type and reconstruct tree view.
There are some basic rules that underlie the implicit type convertion:
  • string is any string except reserved keywords below.
  • number must be a number.
  • boolean must be true or false keyword.
  • object must be wrapped in curly braces {}.
  • array must be wrapped in square brackets [].
  • function must start with function keyword and must have right function definition.

Examples of typing to the input field:
  • Hello World! — convert to String
  • 20.14 — convert to Number
  • true — convert to Boolean
  • {"json": "tree"} — convert to Object
  • [1,2,3] — convert to Array
  • function(){} — convert to Function
