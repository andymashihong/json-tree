edit-level

This attribute is used for specifying the editing level. 
<json-tree json="jsonData" edit-level="high"></json-tree>
edit-level="high"
edit-level="low"
{{jsonData | json}}
By default, it is used a high edit level that allows you to use all operations.
If you want to operate only with key-values of the nodes and to avoid unexpected transformation of json tree, you can add low edit-level attribute.

collapsed-level

This attribute is used for customizing initial depth of tree view. It takes a numeric value, default equals to 0. 
<json-tree json="jsonData" collapsed-level="0"></json-tree>
collapsed-level="0"
collapsed-level="1"
collapsed-level="2"
{{jsonData | json}}

If collapsed-level <= 0, then json-tree is fully collapsed. If collapsed-level == 1, then the first level node would be expanded. If == 2 - the first and second level nodes. And so on.

node

This attribute is used to get access for some internal json-tree API. 
<json-tree json="jsonData" node="nodeOptions"></json-tree>
For example, you can completely refresh directive by using directive internal refresh function: 
$scope.nodeOptions.refresh();
